Preguntas

  • ¿Cuál es la diferencia entre Internet y la World Wide Web?

    • Internet es una inmensa red de computadoras alrededor de todo el mundo conectadas entre sí. En cambio, la web (la World Wide Web) es una enorme colección de páginas que se asienta sobreesa red de computadoras.

  • ¿Cuáles son las partes de un URL?

    • URL significa Uniform Resource Locator. Los elementos de un URL son:

    • HTTPS
    • Subdominio
    • Dominio
    • TLD
    • Ruta
    • Parámetro
    • Etiqueta

  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    • Los métodos HTTP son las acciones que se pueden realizar en un recurso. Los más comunes son GET, HEAD, POST, PUT, PATCH y DELETE. GET: Se utiliza para obtener información de un recurso. HEAD: Se utiliza para obtener los metadatos de un recurso. POST: Se utiliza para enviar información a un recurso. PUT: Se utiliza para actualizar un recurso. PATCH: Se utiliza para actualizar parcialmente un recurso. DELETE: Se utiliza para eliminar un recurso.

  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    • Se debe utilizar POST. Esto se debe a que el formulario contiene información sensible que no debe ser visible para el usuario.

  • ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    • Se utiliza GET. Porque el navegador web envía la información a través de la URL y la URL es un recurso.

  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    • El código de respuesta 200, ok, es un código de respuesta del servidor HTTP que nos ofrecerá el estatus ante una petición estandar correcta a la que puede responder sin problemas. La petición enviada por el cliente, navegador o robot, fue recibida correctamente, puede ser entendida por el servidor y aceptada por éste.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    • Si es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404. Esto se debe a que el error 404 significa que el recurso que se intentó acceder no existe. Por lo que si esto no debería suceder como desarrollador debe arreglarlo.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    • No es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500. Esto se debe a que el error 500 significa que el servidor no pudo procesar la solicitud. Por lo que es un error ajeno al desarrollador.

  • ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    • Los tags y atributos desaprobados son partes del estándar HTML cuyo uso ya no es recomendado. Esto sucede dado que el estándar es actualizado de vez en cuando (por ejemplo, de la versión 3.0 a la 4.0) y alguno tags o características del lenguaje son agregada, removidas o desaprobadas.
    Elementos desaprobados en favor de otros elementos:

    • applet: Reemplazado por el tag HTML object.
    • dir: Reemplazado por el tag HTML ul.
    • isindex: Reemplazado por el tag HTML input.
    • menu: Reemplazado por el tag HTML ul.

    Elementos desaprobados en favor de las hojas de estilo:
    • basefont
    • center
    • font
    • s
    • strike
    • u

  • ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    • HTML5 mejora varios aspectos de HTML4 que estaban obsoletos y eran difíciles de trabajar. Además, HTML5 puede hacer muchas cosas de forma nativa, mientras que HTML4 depende por completo de sistemas que hace tiempo que están en desuso, como Silverlight, Java Web Start y Flash.

  • ¿Qué componentes de estructura y estilo tiene una tabla?

    • Las tablas tienen dos componentes principales: la estructura y el estilo. La estructura de una tabla se define con las etiquetas table, tr, td, th, thead, tbody, tfoot, caption. El estilo de una tabla se define con las etiquetas colgroup, col, style, align, bgcolor, border, cellpadding, cellspacing, frame, rules, summary, width.

  • ¿Cuáles son los principales controles de una forma HTML5?

    • Los principales controles de una forma HTML5 son:

    • input
    • textarea
    • select
    • button
    • output
    • progress
    • meter
    • fieldset
    • legend
    • label
    • form
